## 五：阶段四：状态管理与数据流 (2-3周)

### 5.1 装饰器体系 ⭐

| 装饰器 | 用途 | 使用场景 |
|--------|------|----------|
| @State | 组件内部状态 | 计数器、开关、输入框 |
| @Prop | 父→子单向传递 | 数据展示、只读属性 |
| @Link | 父子双向绑定 | 表单输入、设置项 |
| @Provide/@Consume | 跨层级传递 | 主题、用户信息 |
| @Observed/@ObjectLink | 嵌套对象观察 | 列表项、购物车 |
| @Watch | 监听状态变化 | 数据验证、副作用 |

**学习顺序**:
```
1. @State → 2. @Prop → 3. @Link → 4. @Observed/@ObjectLink → 5. @Provide/@Consume → 6. @Watch
```

**练习**:

- [ ] 图片加载与缓存
- [ ] 请求封装与拦截器
- [ ] 错误处理

**示例**:
```typescript
import http from '@ohos.net.http'

// GET 请求
function getData(url: string): Promise<void> {
  const httpRequest = http.createHttp()
  return httpRequest.request(url, {
    method: http.RequestMethod.GET,
    header: { 'Content-Type': 'application/json' }
  })
}
```

## 十、学习建议

### 10.1 学习方法

1. **理论与实践结合**: 每学一个知识点就动手实践
2. **由简到繁**: 先完成小项目，再做复杂应用
3. **多看官方示例**: 官方 Codelabs 有很多高质量示例
4. **加入社区**: 与其他开发者交流经验

### 10.2 常见误区

| 误区 | 正确做法 |
|------|----------|
| 直接做复杂项目 | 从基础组件和简单应用开始 |
| 忽视 ArkTS 语法限制 | 严格遵循 ArkTS 规范 |
| 不重视状态管理 | 系统学习装饰器体系 |
| 只看文档不写代码 | 多动手实践 |

### 10.3 时间规划建议

| 学习阶段 | 预计时间 | 每周投入 |
